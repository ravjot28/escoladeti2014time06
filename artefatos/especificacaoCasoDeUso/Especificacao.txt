(a) usuárioes:  Administrador.

(b) Pré-condições:
	1. Administrador estar logado no sistema.
	2. Deve haver uma Pessoa Física cadastrada no sistema.

(c) Pós-condições:
	1. O usuário fica habilitado a efetuar login no sistema.
	2. O usuário estará salvo no sistema.

	
Fluxo Básico:
	1.	O sistema exige informações obrigatórias para cadastro de usuário no sistema:
		1.1	Usuário
		1.2	Senha
		1.3	Confirmar Senha
		1.4	CódigoPessoa
	
	2.  Salvar Pessoa 
		2.1	Validar campos obrigatórios
		2.2	Validar senha apartir critérios definidos
			2.2.1	Exige senha com mínimo de 6 caracteres
			2.2.2	Exige senha com letras e números
		2.3	Validar nome de usuário se não existe mais de um no sistema
		2.4 Validar se Pessoa já possui usuário cadastrado
		

Fluxo Alternativo A:
	1. No passo 2.1 do Fluxo Básico, caso haja algum campo obrigatório não preenchido:
	2. O sistema informa o erro ao ator.
	3. O fluxo retorna ao passo 1 do fluxo básico.
Fluxo Alternativo B:
	1. No passo 2.2 do Fluxo Básico, caso verifique que só possuam letras na senha:
	2. O sistema informa senha inválida ao ator.
	3. O fluxo retorna ao passo 1 do fluxo básico.
